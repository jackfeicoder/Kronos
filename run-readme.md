UI 上的这三个核心参数（采样温度、核采样范围、样本数）是控制大模型预测行为的关键。用“大白话”和金融量化的视角来解释，它们的意思及最佳设置如下：

---

### 1. 预测采样温度 (Temperature / T)
* **大白话意思**：控制模型预测的**“想象力”**或**“随机波动幅度”**。
  * **调太低（如 0.2）**：模型会变得极度保守，预测出的走势图容易变成一条平庸的、没有波澜的直线。
  * **调太高（如 1.8）**：模型的想象力会暴增，预测出的振幅巨大，可能出现夸张的暴涨暴跌，但也会夹杂大量毫无根据的随机噪音。
* **推荐数值**：**`0.8` 到 `1.0`**（最合适。既保留了合理的波动，又不会让模型走势彻底失控）。

### 2. 核采样范围 (top_p)
* **大白话意思**：控制模型在预测下一步走势时，**“候选方案库的筛选范围”**。
  * **调太低（如 0.3）**：模型每次只在前 30% 最平庸稳妥的可能里选择，预测出的趋势会非常单调和套路化。
  * **调高（如 0.9）**：模型会把 90% 的合理可能性都纳入考虑，能够捕捉到一些看似是“小概率”但符合金融逻辑的局部反转和突破走势。
* **推荐数值**：**`0.9`**（这是时序大模型和学术界的黄金默认值，表现最均衡）。

### 3. 生成样本路径数 (Sample Count)
* **大白话意思**：点击预测时，模型在后台**生成多少条“平行宇宙”的走势曲线，并求均值作为最终结果输出**。
  * **设为 1**：只生成一条，计算最快。但因为模型有随机性，这一条曲线可能会带有偶然的“运气成分”（比如刚好采到了一条偏极端的路径）。
  * **设为 3 ~ 5**：模型在后台进行 3 到 5 次模拟，并把结果平均起来。这能非常有效地**消除单次采样的随机毛刺（噪声），得出在统计学上最稳健、最可能的趋势**。
* **推荐数值**：
  * **`3` 或 `5`**（如果您配有显卡，或者 CPU 性能较好，强烈建议设为 3 或 5，这样得到的平均预测曲线最具有交易参考价值）。
  * 如果您感觉运行有点卡，可以设为 `1` 以获得最畅快的响应速度。

---

### 💡 黄金参数组合推荐（抄作业指南）

| 模式 | 采样温度 (T) | 核采样 (top_p) | 样本路径数 (Sample Count) | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **稳健参考模式（推荐 ⭐）** | **`0.8`** | **`0.9`** | **`3`** 或 **`5`** | **最适合用来辅助判断买卖点**，曲线平滑且过滤了大部分噪声。 |

| **保守大势模式** | **`0.5`** | **`0.9`** | **`3`** | **只想看最平稳的主线趋势**，几乎完全过滤掉日内的上下插针波动。 |
